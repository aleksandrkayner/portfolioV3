# AI / Agent navigation guide

Use this file to orient quickly when fixing bugs or adding features.

## Entry points

| File | Role |
|------|------|
| `src/main.tsx` | React bootstrap |
| `src/App.tsx` | Renders `<Dashboard />` only |
| `src/components/layout/Dashboard.tsx` | Page shell + intro copy |
| `src/config/widgets.ts` | **Widget registry** — start here for new widgets |
| `src/data/portfolio.ts` | **Content** — start here for copy/data changes |

## Common tasks

### Change text, links, or project data

→ `src/data/portfolio.ts`

### Add a new widget

1. `src/components/widgets/NewWidget.tsx` — use `WidgetShell`, accept `WidgetComponentProps`
2. Register in `src/config/widgets.ts` with `id`, `grid: { w, h }`, `order`, `component`
3. Optionally add slot in `TEMPLATE_3COL` in `src/config/gridLayout.ts`
4. Update `docs/WIDGETS.md`

### Hide a widget without deleting code

→ Set `enabled: false` on its entry in `src/config/widgets.ts`

### Fix grid layout

→ `src/config/gridLayout.ts` (`WIDGET_PLACEMENTS`, `COL_MIN_PX`, `ROW_HEIGHT_PX`) 
→ `src/components/layout/DashboardGrid.tsx` (react-grid-layout, draggable)

### Styling / theme

→ `src/index.css` (`[data-theme]` CSS variables)  
→ `src/stores/themeStore.ts`, `src/di/container.ts`, `src/components/ui/ThemeToggle.tsx`  
→ See `docs/THEMES.md`

### State (MobX + tsyringe)

→ `src/di/container.ts` — register and resolve stores  
→ `src/stores/themeStore.ts`, `src/stores/layoutStore.ts`

## Do not

- Hard-code portfolio copy inside widget components — use `portfolio.ts`
- Add widgets only in JSX without registering in `widgets.ts`
- Edit `dist/` — it is generated by `npm run build`

## Build verification

```bash
npm run build && npm run preview
```

## Prompt history

**Always** add a row to `docs/PROMPT-LOG.csv` after every user prompt (all columns), then run `npm run prompt-log:export` to refresh `docs/PROMPT-LOG.xlsx`.
